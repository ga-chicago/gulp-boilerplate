Introducing Gulp

Welcome to Gulp

Gulp is a task runner that makes common boring tasks easy and efficient. There are two main task runners in JavaScript: Grunt and Gulp. Neither one is better than the other. They only differ in how they operate. There are many more task runners for JavaScript out there than just Grunt and Gulp but we'll focus only on Gulp today.

Gulp's differentiating feature is that is lets you write tasks using vanilla JavaScript. It operates mostly on "pipes" which basically means that the output from one program is fed to another program before any output is given. You can chain programs together so one input, like say a JavaScript file, can be piped through an almost infinite number of programs in sequence to do things like check for errors, enforce code styles, minification, etc.

Using Gulp

The page you are reading now is made possible by the power of Gulp. The basic idea is that you install the Gulp command line application globally, then install the gulp library and any plugins locally to your project. With all of your dependencies installed you now just need to apply transformations to files through Gulp tasks.

Gulp tasks are written in a 

gulpfile
which is a vanilla JavaScript file. See this project's gulpfile for an example of this in action.

Source, Destination, Watch

With Gulp you'll use three main parts of its API. The 

src
, 
dest
, and 
watch
methods. The 'src' method tells gulp to look for a file(s)/folder(s) and treat what it finds as source material then pipe it to the next task. After being piped through multiple functions your code will likely be ready to be released to its destination. This is where the 'dest' method comes in. This method tells gulp where to put the changed output of the piped data. Gulp's 'watch' method simply watches files or folders then executes tasks whenever changes are made to them. This allows for cool tricks like not having to use Nodemon, Forever, or other libraries to monitor your code.

What can you do with Gulp tasks?

Here's the short list:

  • Run a static server
  • Run a web application server
  • Lint JavaScript files
  • Transpile LESS to CSS
  • Reload pages when files change
  • Replace references to local libraries with those hosted on a CDN
  • Concatenate files together
  • ...and much more
